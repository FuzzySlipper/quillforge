## Scenario 1: Lore Lookup vs Creative Writing

> "Write me a paragraph about the history of the Ironclad Mountains."

### 1. Initial Assessment
The request contains two distinct operations bundled into one imperative: *writing* and *history*. "History" implies established world facts that may already exist in the lore set. "Write me a paragraph" implies prose generation. The ambiguity is whether the Ironclad Mountains exist in the active lore set ("builder") or need to be invented. Inventing lore without checking would risk contradicting established canon — exactly the continuity problem an editor persona should flag first.

### 2. Tool Selection
**Step 1:** `query_lore` — query: `"Ironclad Mountains history"`
Reason: Check whether canonical facts exist before generating prose. The active lore set is "builder," so results are grounded there.

**Step 2 (conditional on Step 1 returning results):** `write_prose` — scene_description would incorporate the lore query results, tone_notes might default to "historical/expository."

**Step 2 (conditional on Step 1 returning nothing):** I'd flag to the user that no lore exists and ask whether to invent canonical history or note it as provisional. I would *not* silently fabricate and present it as lore-grounded.

### 3. Tools Considered but Rejected
- **`web_search`** — Rejected immediately. The Ironclad Mountains are fictional; real-world search results are irrelevant and would produce noise or misleading analogues.
- **`write_prose` first** — Rejected as the *opening* move. Writing before checking lore inverts the correct workflow for a continuity-focused editor persona.

### 4. Mode Boundaries
Fully within scope for General mode. The routing logic maps directly: lore question → `query_lore`, then creative writing → `write_prose`. No redirection needed.

### 5. Assumptions
- The Ironclad Mountains are part of the "builder" world, not a reference to something external.
- "Paragraph" means polished, publishable prose — not a bullet-point summary — per the editor persona's clean-output standard.
- The user wants lore-accurate output, not exploratory invention.
